# Additional people

Multi-person households: each additional person gets `people/<slug>/`
mirroring the root `journal/` structure (profile.md, history.md,
training.md, goals.md, gear.md, health.md, entries/, plans/). The root
`journal/` is the default person.

Every data structure carries a `person` id; the dashboard shows a person
switcher when more than one profile exists. Each person's MCP connections
(their Garmin/Strava/etc. accounts) are their own.
